CEH Module 4 — Enumeration Theory STEP 3 IN THE ETHICAL HACKING METHODOLOGY

Enumeration is the process of extracting detailed information from a target after scanning. Unlike scanning (which identifies open ports), enumeration actively connects to services to pull usernames, service versions, OS details, and configuration data.

Technique

Banner Grabbing

Capturing service banners to identify software version and OS. Done via Netcat, Telnet, or Nmap NSE scripts. Reveals exact version numbers for CVE matching.

Technique

SNMP Enumeration

Port 161/UDP. Default community string "public" exposes network devices, processes, and open ports. Tool: snmpwalk.

Technique

LDAP Enumeration

Port 389. Used by Active Directory. Can reveal usernames, groups, and organizational structure. Tool: ldapsearch.

Technique

NetBIOS / SMB

Ports 137-139, 445. Reveals Windows computer names, logged-in users, and network shares. Tools: nbtscan, enum4linux, smbclient.

Key Takeaways WHAT I LEARNED ON DAY 6
✓

Enumeration reveals the attack surface

Banner grabbing identified vsftpd 2.3.4 which directly led to the CVE-2011-2523 exploit. Without enumeration, I wouldn't know which version to target.

✓

Metasploit is a framework, not magic

Understanding search, use, show options, set, info, and run — and knowing WHY each option exists — matters more than just running modules blindly.

✓

Unpatched software = root access

The target OS is from 2008. One unpatched FTP daemon gave full root in under 10 seconds. This is why patch management is the #1 security control.

✓

Documentation is as important as execution

Every screenshot, every command output, every CVE reference — this is what separates a professional penetration tester from someone just running tools.
